# Command prd

Le PRD (Product Requirement Document) est un document qui décrit les fonctionnalités, les exigences et les spécifications d'un produit ou d'une fonctionnalité à développer. Il sert de guide pour les équipes de développement, de design et de gestion de projet afin de s'assurer que tous les membres comprennent les objectifs et les attentes du projet.

Génère un document markdown dans doc/prd qui décrit les fonctionnalités, les exigences et les spécifications du produit ou de la fonctionnalité à développer. Le document doit inclure une description claire du problème à résoudre, des objectifs du projet, des fonctionnalités clés, des exigences techniques et des critères de succès.
